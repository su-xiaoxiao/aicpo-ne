# 文档目录说明

## 目录结构

### `00-background`

存放项目背景基线、范围说明、术语基线等长期稳定输入。

### `01-governance`

存放建模规范、命名规范、关系规范、评审规则、输出模板等治理类文档。

当前建议包含：

- 仓库级协作规则摘要
- 命名规范
- 关系建模规范
- 输出规范
- 评审规范

当前已建立基础文档：

- `00-AGENTS摘要.md`
- `01-命名规范.md`
- `02-关系建模规范.md`
- `03-建模输出规范.md`

### `02-domain`

存放领域概念、业务对象说明、角色定义、核心术语扩展说明。

### `03-architecture`

存放总体架构、数据链路、决策链路、分层说明、集成边界说明。

当前建议包含：

- 总体链路设计
- 数据链路设计
- 决策链路设计
- 分层职责设计
- 边界与依赖设计

### `05-templates`

存放各类标准文档模板，供背景、治理、领域、架构、模块、模型、模型图、评审记录复用。

### `10-modules`

按模块存放设计文档。建议一个模块一个子目录，目录内包含该模块的概念、逻辑、物理设计及说明。

建议命名方式：

- `10-modules/<模块编码>-<模块名称>/`

当前建议包含：

- 首批模块拆分清单
- 各模块设计目录

### `20-models/concept`

存放概念模型文档。

### `20-models/logical`

存放逻辑模型文档。

### `20-models/physical`

存放物理模型文档。

### `30-diagrams/concept`

存放概念模型图。

### `30-diagrams/logical`

存放逻辑模型图。

### `30-diagrams/physical`

存放物理模型图。

### `40-reviews`

存放模型校验、设计评审、问题清单、待决事项。

### `90-archive`

存放废弃版本、历史快照、归档文档。

## 文件命名建议

建议统一采用以下命名风格：

- `NN-主题名称.md`
- `NN-模块名称-概念模型.md`
- `NN-模块名称-逻辑模型.md`
- `NN-模块名称-物理模型.md`

模型图建议与文档同名前缀保持一致，便于追踪。
